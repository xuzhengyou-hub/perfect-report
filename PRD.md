# 智能实验报告生成工具 PRD

## 1. 产品目标

本产品用于根据实验资料和学校提供的实验报告模板，生成可直接提交或少量补充后提交的实验报告 `.docx` 文件。

当前版本的核心目标不是“生成一篇格式漂亮的新文档”，而是：

1. 严格保留用户上传模板的原始版式。
2. 将可确认的信息填入模板中的对应位置。
3. 对无法确认的字段明确标记为 `【待用户填写：字段名】`。
4. 支持结合文本、PDF、图片等资料辅助填写实验内容。

## 2. 目标用户

1. 需要按照固定学校模板提交实验报告的学生。
2. 有实验资料但不想手动整理和回填模板的课程使用者。
3. 需要减少报告整理时间、同时又不接受版式跑偏的用户。

## 3. 核心问题

旧版本存在一个关键偏差：

- 它把模板当成“写作参考提纲”
- 再由模型输出整篇 markdown
- 最后重新生成一份新的 Word 文档

这会导致：

1. 模板封面、页眉、表格结构和空白位置被破坏。
2. 内容位置与学校模板不一致。
3. 即使内容合理，最终交付物依然不符合“严格套模板”的真实需求。

## 4. 当前产品原则

### 4.1 模板优先

上传的 `.docx` 模板就是最终版式来源。系统不得在最终输出阶段重建一个自由版报告。

### 4.2 原位填充

系统应把模板识别为一张待填写表单，优先填充：

1. 首页封面表格字段
2. 第二页信息表格字段
3. 正文锚点段落后的内容区
4. 模板指定的图示插入区域

### 4.3 缺失显式化

对缺失信息，系统必须输出明确占位，不允许臆造：

- `【待用户填写：班级】`
- `【待用户填写：学号】`
- `【待用户填写：指导教师签名】`

### 4.4 调试产物与最终产物分离

`final_report.md` 可以继续保留，作为中间预览或调试产物，但不能再主导最终 `.docx` 版式。

## 5. 输入与输出

### 5.1 输入

1. 实验题目
2. 用户补充说明
3. 报告模板 `.docx`
4. 背景资料文件
   - `.txt`
   - `.pdf`
   - `.png`
   - `.jpg`
   - `.jpeg`

### 5.2 输出

主输出：

1. `result_report.docx`

辅助输出：

1. `parsed_background.md`
2. `synthesized_knowledge.md`
3. `final_report.md`
4. `report_structured.json`

其中：

- `result_report.docx` 是正式交付物
- 其余文件是调试、回溯和开发辅助产物

## 6. 功能范围

### 6.1 当前已支持

1. 上传模板与背景资料
2. 后端异步任务执行
3. 进度推送与下载
4. PDF 文本与图片提取
5. 图像摘要辅助理解
6. 知识综合
7. 严格模板填充
8. 缺失字段占位

### 6.2 当前模板填充能力

已按现有实验报告模板支持：

1. 封面表格字段填充
   - 课程名称
   - 实验项目
   - 实验时间
   - 实验班级
   - 总份数
   - 指导教师
2. 第二页信息表格字段填充
   - 学院
   - 专业
   - 班级
   - 成绩
   - 姓名
   - 学号
   - 组别
   - 组员
   - 实验地点
   - 实验日期
   - 指导教师签名
3. 考核表格字段填充
   - 预习情况
   - 操作情况
   - 考勤情况
   - 数据处理情况
4. 正文区段填充
   - 实验目的
   - 实验内容
   - 实验步骤
   - 序列图组成
   - 协作图组成
   - 事件流分析
   - 序列图基本思路
   - 协作图操作步骤
   - 实验小结
5. 图片插入
   - 序列图
   - 协作图

## 7. 非目标

当前版本不追求：

1. 自动适配任意未知结构的 Word 模板
2. 完整的通用 Word 模板引擎
3. 百分之百自动补全所有学生信息
4. 基于 markdown 的通用文档美化能力

## 8. 成功标准

### 8.1 功能成功标准

1. 输出文档保持原模板主要版式不变。
2. 已知信息出现在正确位置。
3. 未知信息显示为明确占位，而不是空想内容。
4. 用户可以在生成后快速定位并手动补齐剩余字段。

### 8.2 工程成功标准

1. 后端测试通过。
2. 核心模板填充逻辑具备回归测试。
3. 中间产物可追踪，便于定位生成问题。

## 9. 后续规划

1. 增加模板字段映射配置层，支持多模板切换。
2. 引入更细粒度的图片槽位定位。
3. 为“待用户填写”字段生成前端待补清单。
4. 增加生成后文档预览与差异检查。
